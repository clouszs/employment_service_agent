## 一、当前前端现状

### 1.1 已有页面文件

所有静态页面统一存放在 `backend/smart-career-platform/pages/` 下：

| 页面文件 | 用途 | 角色 |
|----------|------|------|
| `login.html` | 登录页（深蓝科技风，学号/工号 + 密码） | 全部 |
| `chat.html` | AI 智能问答（侧边栏会话列表 + 主对话区 + 引用卡片 + 置信度 badge） | 学生/老师/管理员 |
| `conversations.html` | 会话历史（卡片列表 + 搜索/筛选/分页 + 置信度条） | 学生/老师/管理员 |
| `admin-dashboard.html` | 管理后台数据总览（KPI 卡片 + 趋势图 + 告警 + 快捷统计） | admin/editor |
| `admin-faq.html` | FAQ 问答对管理（表格 + 批量操作栏 + 新建/编辑弹窗 + 启用开关） | admin/editor |

### 1.2 前端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 页面形式 | 纯静态 HTML（非 SPA） | 每个页面独立文件，通过 file:// 或 nginx 直接打开 |
| CSS 方案 | Tailwind CSS v4（CDN browser）+ 自定义 CSS 变量 | 深蓝科技风设计令牌已统一 |
| 图标 | Lucide Icons（UMD CDN） | 统一图标风格 |
| JS 交互 | 原生 JavaScript（无框架） | fetch / EventSource / localStorage |
| 认证 | localStorage 存 `access_token` | 手动加 `Authorization` header |
| 后端地址 | 硬编码或配置文件注入 `API_BASE` | 建议统一在 `<script>` 顶部配置 |

### 1.3 当前缺失的后端对接

| 页面 | 缺失对接 |
|------|----------|
| `login.html` | 登录表单未调用 `/auth/login`；Token 未存储；无路由跳转 |
| `chat.html` | 发送按钮未调用问答接口；引用卡片未对接 `/references`；流式未实现 |
| `conversations.html` | 列表/搜索/筛选/删除均为静态，未调用 `/conversations` 接口 |
| `admin-dashboard.html` | 所有 KPI 数字为静态 mock，未调用 `/stats`、`/kb-health`、`/llm-cost` |
| `admin-faq.html` | 表格数据为静态 mock，新建/编辑弹窗未调用 `/faqs` 接口 |

## 六、角色与权限

### 6.1 角色定义

后端实际采用两层权限体系：

| 前端标签 | 对应 role_code | 对应 user_type | 可用功能 |
|----------|---------------|---------------|----------|
| 学生 | `student` | 1/2 | 对话问答、查看自己的会话/消息 |
| 老师 | `editor` | 3/4 | 对话问答 + 知识库管理 + 运营中心 |
| 管理员 | `admin` | 5 | 全部功能 + 用户管理 + 系统设置 + 监控中心 |

> **注意**：后端没有独立的 "HR" 角色编码。辅导员(user_type=3) 和老师(user_type=4) 在 RBAC 层面共用 `editor` 角色，权限相同。若业务上需要区分，需先在后台 `sys_role` 表中新增独立角色编码。

### 6.2 前端权限控制

- **Token 存储**：`localStorage.setItem('access_token', token)`
- **请求拦截**：所有 fetch 请求统一加 `Authorization: Bearer <token>` header
- **UI 隐藏**：根据 `/auth/me` 返回的 `roles` 数组隐藏无权限的菜单项
- **路由守卫**：页面加载时检查 token 有效性，无效则跳转 `login.html`
- **后端鉴权**：前端仅做 UI 隐藏，所有接口仍以后端 `require_roles` 为准

## 八、关键接口对接代码示例

### 8.1 登录对接

```javascript
// login.html 登录按钮事件
async function handleLogin() {
  const username = document.querySelector('#username').value;
  const password = document.querySelector('#password').value;
  const resp = await fetch('/api/v1/auth/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ username, password }),
  });
  const result = await resp.json();
  if (result.success) {
    localStorage.setItem('access_token', result.data.access_token);
    const roles = result.data.user.roles || [];
    if (roles.includes('admin') || roles.includes('editor')) {
      window.location.href = '/pages/admin-dashboard.html';
    } else {
      window.location.href = '/pages/chat.html';
    }
  } else {
    alert(result.message || '登录失败');
  }
}
```
